# Design Documents

This directory contains **technical design documents** and **API specifications** generated by the **Design Agent** (`02-design-agent`).

## Structure

Each feature has a design document following the naming convention:
```
DESIGN-{FEATURE-NAME}-API.md
```

Corresponding OpenAPI specifications are placed in:
```
src/main/resources/api/{feature}-api.yaml
```

## Contents

| Document                       | Feature         | Status     |
|-------------------------------|-----------------|------------|
| [DESIGN-PRODUCT-CATALOG-API.md](DESIGN-PRODUCT-CATALOG-API.md) | Product Catalog | ✅ Complete |

## How Designs Are Created

The Design Agent:
1. Reads the Epic and User Stories from `docs/planning/`
2. Designs REST API endpoints with request/response schemas
3. Creates an OpenAPI 3.0 specification
4. Documents error handling, pagination, and validation rules
