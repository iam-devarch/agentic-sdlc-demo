## Epic Title
Product Catalog REST API

## Business Context
The application needs a Product Catalog service that allows consumers to browse, search, and manage products. This is a core microservice capability that other services and front-end applications depend on.

## Goals
- Provide a RESTful API for CRUD operations on products
- Support pagination for listing products
- Validate all input data
- Return consistent error responses

## Scope

**In Scope:**
- Create, Read, Update, Delete operations for products
- List products with pagination
- Input validation
- Error handling

**Out of Scope:**
- Product categories/taxonomy
- Image upload
- Inventory management
- Authentication/authorization

## Success Criteria
- All CRUD endpoints functional and returning correct HTTP status codes
- Input validation prevents invalid data
- Errors return structured JSON responses
- Unit tests cover all layers with passing results

## User Stories

### US-1: List All Products

**As a** API consumer  
**I want to** retrieve a paginated list of products  
**So that** I can display products to end users  

**Acceptance Criteria:**
- [ ] GET `/api/v1/products` returns a list of products
- [ ] Supports `page` (default: 0) and `size` (default: 20) query parameters
- [ ] Returns 200 OK with product list
- [ ] Empty list returns 200 OK with empty array

**Priority**: High  
**Size**: M

---

### US-2: Get Product by ID

**As a** API consumer  
**I want to** retrieve a single product by its ID  
**So that** I can display product details  

**Acceptance Criteria:**
- [ ] GET `/api/v1/products/{id}` returns the product
- [ ] Returns 200 OK with product data
- [ ] Returns 404 Not Found if product doesn't exist

**Priority**: High  
**Size**: S

---

### US-3: Create a New Product

**As a** API consumer  
**I want to** create a new product  
**So that** I can add products to the catalog  

**Acceptance Criteria:**
- [ ] POST `/api/v1/products` creates a new product
- [ ] Request body must include `name` (required, 1-255 chars) and `price` (required, > 0)
- [ ] `description` is optional (max 1000 chars)
- [ ] Returns 201 Created with the created product
- [ ] Returns 400 Bad Request if validation fails

**Priority**: High  
**Size**: M

---

### US-4: Update an Existing Product

**As a** API consumer  
**I want to** update an existing product  
**So that** I can modify product information  

**Acceptance Criteria:**
- [ ] PUT `/api/v1/products/{id}` updates the product
- [ ] Request body validation same as creation
- [ ] Returns 200 OK with updated product
- [ ] Returns 404 Not Found if product doesn't exist
- [ ] Returns 400 Bad Request if validation fails

**Priority**: Medium  
**Size**: M

---

### US-5: Delete a Product

**As a** API consumer  
**I want to** delete a product by its ID  
**So that** I can remove products from the catalog  

**Acceptance Criteria:**
- [ ] DELETE `/api/v1/products/{id}` removes the product
- [ ] Returns 204 No Content on success
- [ ] Returns 404 Not Found if product doesn't exist

**Priority**: Medium  
**Size**: S
