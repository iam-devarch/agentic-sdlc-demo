# Planning Documents

This directory contains **epic-level requirements** and **user stories** generated by the **Planning Agent** (`01-planning-agent`).

## Structure

Each feature has an epic document following the naming convention:
```
EPIC-{FEATURE-NAME}.md
```

## Contents

| Document                       | Feature         | Status     |
|-------------------------------|-----------------|------------|
| [EPIC-PRODUCT-CATALOG.md](EPIC-PRODUCT-CATALOG.md) | Product Catalog | ✅ Complete |

## How Epics Are Created

The Planning Agent:
1. Receives a high-level feature request
2. Creates an Epic with business context, goals, and scope
3. Breaks the Epic into User Stories with acceptance criteria
4. Each User Story is sized and prioritized
